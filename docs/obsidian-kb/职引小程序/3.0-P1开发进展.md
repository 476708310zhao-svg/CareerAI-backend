# 职引小程序 3.0 P1 开发进展

日期：2026-07-06

## P1 目标

在 P0 的工作台、JD 匹配、投递追踪、校招筛选和我的页资产总览之上，继续增强用户长期使用能力，重点包括简历版本管理、面经 AI 回答与 STAR 生成、AI 模拟面试复盘、校招订阅提醒增强和会员次数限制。

## 已完成：简历版本管理

1. 简历页当前版本卡片展示版本名称、默认状态、目标岗位、更新时间、完整度和优化记录数量。
2. 简历版本管理面板展示每份简历的默认标识、目标岗位和更新时间。
3. 支持编辑当前简历版本名称和目标岗位，保存后同步后端 `/api/resumes/:id`。
4. 支持从当前版本卡片直接设为默认简历，复用后端 `/api/resumes/:id/default`。
5. AI 润色采纳后，优化历史会即时显示在简历页，保留最近 5 条优化摘要。

## 已完成：面经题库 AI 训练

1. 面试题详情页新增 AI 面试训练面板。
2. 支持三种生成模式：标准回答、STAR 答案、模拟追问。
3. 生成时会结合题目、分类、难度、参考答案、页面面试要点和本地在线简历背景。
4. AI 训练稿支持复制。
5. AI 训练稿支持保存到面试错题本，便于后续每日练习。
6. AI 请求不可用时自动生成本地兜底训练稿，不阻断用户练习。

## 已完成：JD 匹配与网申助手简历版本统一

1. 新增共享简历版本工具，优先读取服务端默认简历，接口不可用或未登录时自动回退本地在线简历。
2. JD 快速匹配页支持选择简历版本，匹配报告记录所用 `resumeName` 和 `resumeVersionId`。
3. JD 匹配结果加入投递追踪时同步带入简历版本 ID。
4. AI 求职助手的网申助手面板支持选择简历版本，生成草稿时结合选中简历、岗位和题干。
5. 申请材料记录保留 `resumeName` 和 `resumeVersionId`，材料库列表展示“基于哪份简历”。
6. 后端 JD 匹配报告和申请材料表补充简历版本字段，兼容旧数据。

## 已完成：AI 模拟面试复盘增强

1. 修复模拟面试总结 JSON 解析问题，报告阶段可正确读取 `totalScore`、`dimensions`、`qaScores` 等结构化字段。
2. 面试结束报告新增每题评分、薄弱题、行动清单、下次练习建议和风险提醒。
3. 报告页新增“下一步训练清单”和“薄弱题”模块，支持一键加入错题本。
4. 评分兜底从随机浮动改为确定性规则，避免同一份回答重复生成时分数不稳定。
5. 复盘中心记录卡展示薄弱维度、薄弱题数量和下一步动作。

## 已完成：校招订阅提醒增强

1. 新增校招截止提醒共享工具，列表页和详情页统一使用 `campus_schedule + campus_deadline` 提醒记录。
2. 校招列表“提醒”按钮支持设置截止提醒、同步服务端提醒状态，并可从已提醒状态取消提醒。
3. 校招详情页提醒卡片改为真实截止提醒设置，文案明确为截止前 7/3/1 天和当天提醒；截止日期缺失时降级为本地关注。
4. 后端通用提醒接口支持 `campus_deadline` 类型，并预留 `campus_open` 类型。
5. 提醒调度命中校招截止日期时写入 `campus_deadline` 站内消息，并复用微信订阅消息模板发送。
6. 旧版 `/api/notify/campus-subscribe` 入口保持兼容，同时写入通用提醒队列。

## 已完成：会员次数限制

1. 统一后端 AI 额度账本，普通用户每日额度由 `utils/aiQuota.js` 集中维护，VIP 用户不限次数。
2. 新增 `/api/ai/usage` 查询接口，返回当天各 AI 功能的已用次数、上限、剩余额度和 VIP 状态。
3. AI 求职助手、通用 AI Chat、ATS 简历优化、AI 求职规划、AI 项目生成器和 Networking 消息生成全部接入服务端次数限制。
4. AI 工作流继续保持 VIP 专属，错误返回统一带 `vipRequired` 和功能标识。
5. 会员页新增“今日 AI 额度”模块，展示普通用户剩余额度或会员不限次数状态。
6. 补充冒烟测试，验证额度查询和免费额度用尽时在调用上游 AI 前返回升级提示。

## 已完成：上线配置本地复核

复核日期：2026-07-07

1. 会员支付：本地 `/api/payment/config` 返回 `enabled=true`、`provider=virtual`、`configured=true`、`mock=false`、`available=true`，月卡、季卡、年卡和试用计划均可识别；当前走小程序虚拟支付，不依赖普通微信支付商户参数。
2. 会员开关：`.env` 中 `MEMBERSHIP_FEATURE_ENABLED=true`，但本地数据库 `feature_flags.membership=0`，接口 `/api/features` 返回会员关闭；正式环境发布前需要在管理后台或数据库打开会员开关。
3. 订阅消息：`/api/notify/templates` 可返回代码默认模板 ID，但本地 `.env` 未配置 `WX_TPL_*`；上线前需在微信公众平台确认这些模板 ID 属于当前小程序，并建议写入正式 `.env`。
4. 提醒调度：带 `X-Cron-Secret` 的本地提醒派发接口可正常执行，仓库部署脚本包含 `10 8 * * *` 的提醒派发 cron；正式环境仍需在服务器执行 `crontab -l` 确认任务已注册，并配置强 `CRON_SECRET`。
5. AI 服务：当前 `AI_PROVIDER=ark`，一次最小 AI 健康检查请求返回成功，说明密钥和额度当前可用；正式环境仍需监控上游额度余额和失败率。
6. 生产环境变量：按 `NODE_ENV=production` 校验，本地配置仍缺少或不合格的项包括 `NODE_ENV`、生产强 `JWT_SECRET`、强 `ADMIN_PASSWORD`、`CRON_SECRET`、`WEBHOOK_SECRET`；这些是正式启动前必须补齐的项。
7. `check:data` 的 4 个 warning 已确认均为非阻断上线提示：开发/模拟支付提示文案、服务端 IP 兜底、会员页开发支付 loading 文案、个人页“待完善”空状态文案。

## 已完成：3.0 验收机器人升级

1. 新增 `scripts/e2e-3.0-bot.js`，基于微信开发者工具自动化能力验证 3.0 页面主链路。
2. 新增命令 `npm run bot:e2e-3.0`，覆盖首页、模拟登录、附件简历上传、AI 简历润色、JD 匹配、投递追踪详情、面经 AI 训练、AI 面试报告、校招提醒和会员额度页。
3. 新增命令 `npm run check:e2e-3.0`，用于不启动微信开发者工具的环境 dry-run 检查。
4. 机器人会 mock 微信授权、文件选择、订阅消息、上传和后端请求，避免污染生产数据；真实手机上的登录授权、手机文件选择器、订阅授权和虚拟支付弹窗仍需人工确认。
5. 本机已通过 `check:e2e-3.0` dry-run；真实 E2E 依赖微信开发者工具 `设置 > 安全设置 > 服务端口`。当前本机检测到 `security.enableServicePort=false`，脚本已补充前置检测，避免长时间等待不可连接的 automation 端口。
