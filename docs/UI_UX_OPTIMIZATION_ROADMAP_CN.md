# 求职小程序 UI/UX 产品级优化路线图

> 目标：把小程序逐步打磨成更现代、更统一、更高级、更符合 AI SaaS 商业化产品气质的移动端产品。

## 优化原则

- 移动端优先：优先保证微信开发者工具和真机体验。
- 高级感优先：白底、轻边框、克制阴影、减少廉价渐变。
- 转化优先：让用户清楚知道下一步做什么。
- AI 感知优先：AI 等待、生成、推荐、结果展示要显得专业、聪明、可信。
- 可维护优先：先统一 Design System 和公共组件，再逐页收敛历史样式。
- 性能优先：减少不必要渲染、图片懒加载、长列表和 loading 策略优化。

## 阶段一：全局设计系统与组件统一

- [x] 建立 2026 Product Design System：主色、辅助色、字号、间距、圆角、阴影、动效 token。
- [x] 关闭自动暗色模式，保证当前白色 AI SaaS 风格稳定。
- [x] 统一公共按钮组件 `c-button`。
- [x] 统一公共卡片组件 `c-section-card`。
- [x] 统一搜索框组件 `c-search-bar`。
- [x] 统一标签组件 `c-tag`。
- [x] 统一底部操作栏组件 `c-action-bar`。
- [x] 优化空状态组件 `empty-state`。
- [x] 优化通用 loading 组件 `loading-spinner`。
- [x] 优化 AI loading 组件 `c-ai-loading`。

## 阶段二：性能与首屏体验优化

- [x] 非首屏图片补充 `lazy-load`。
- [x] 保留首页 Banner、用户头像、公司/职位详情主 logo 等首屏关键图即时加载。
- [x] 长列表页检查骨架屏和分页策略。
- [x] 高频页面减少重复请求和重复 setData。
- [x] Banner、公司 logo、资讯图片统一走 HTTPS 与缓存策略。

## 阶段三：核心页面视觉收敛

- [x] 首页：统一金刚区、热门企业、热门岗位、资讯卡片视觉层级。
- [x] 岗位推荐页：统一筛选、列表、地图模式、空状态。
- [x] 职位详情页：继续优化 CTA、AI 面试、求职规划入口。
- [x] 公司详情页：继续优化 tabs、职位列表、AI 洞察展示。
- [x] 题库页：统一顶部 tabs、筛选 chips、题目卡片。
- [x] 测评页：统一机构卡片、对比入口、AI 测评入口。
- [ ] 我的页：统一会员转化、工具列表、设置入口。

## 阶段四：AI 产品体验专项

- [ ] AI 助手继续贴近 ChatGPT 风格：消息布局、输入框、历史侧栏。
- [ ] AI 生成过程加入更专业的阶段反馈。
- [ ] AI 推荐卡片统一为可行动的结果卡。
- [ ] AI 结果页增加复制、保存、继续追问、转化入口。
- [ ] AI 等待状态统一为轻量、可信、不过度动画。

## 阶段五：商业化与转化路径

- [ ] 新用户首屏引导：明确“我能帮你做什么”。
- [ ] 会员转化 CTA：降低打扰，提高关键场景曝光。
- [ ] 简历、岗位、面试、规划形成闭环路径。
- [ ] 空状态转化：空列表不只展示“暂无”，要给下一步行动。
- [ ] 每个核心页面保留一个主 CTA，避免选择过载。

## 阶段六：代码结构与维护

- [ ] 梳理重复页面样式，逐步替换为全局 token。
- [ ] 抽离列表卡片、AI 结果卡、公司 logo、岗位标签等公共组件。
- [ ] 清理历史乱码注释和过期样式覆盖。
- [ ] 建立页面级 UI 验收清单。
