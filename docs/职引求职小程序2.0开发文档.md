# 职引求职小程序 2.0 开发文档

## 1. 文档目标

本文档用于承接《职引求职小程序 2.0 功能规划》，作为后续开发、验收、上线和运营协作的统一参考。

2.0 阶段的核心方向不是继续堆页面，而是把小程序从“求职信息展示工具”升级为“AI 求职管理助手”，让用户能在同一个产品里完成找方向、找岗位、改简历、做网申、练面试、追进度和复盘。

## 2. 产品定位

> 职引是一款面向留学生、应届生和求职者的 AI 求职工作台。

核心价值：

- 帮用户明确自己适合什么岗位。
- 帮用户找到更匹配的岗位和校招机会。
- 帮用户提升简历与岗位 JD 的匹配度。
- 帮用户管理投递、截止、面试和 Offer 进度。
- 帮用户完成网申材料、模拟面试和面试错题复盘。
- 帮运营沉淀岗位、面经、题库、申请材料和用户行为数据。

## 3. 2.0 核心目标

### 3.1 用户目标

- 能系统管理求职进度。
- 不错过岗位、校招和面试关键时间。
- 能根据 JD 优化简历和申请材料。
- 能用 AI 进行模拟面试和复盘。
- 能形成每日求职节奏，而不是临时想起来才打开小程序。

### 3.2 产品目标

- 从“信息浏览”升级为“求职管理系统”。
- 从“单次工具使用”升级为“长期求职工作台”。
- 从“内容展示”升级为“内容 + 工具 + AI 服务 + 会员权益”。
- 为虚拟支付会员体系、求职服务转化和私域运营做准备。

### 3.3 运营目标

- 提高次日留存和 7 日留存。
- 提高岗位收藏率和求职进度录入率。
- 提高简历填写/上传率。
- 提高 AI 功能使用率。
- 提高分享、订阅消息和私域导流转化。

## 4. 三大产品闭环

### 4.1 岗位闭环

```text
找岗位 -> 收藏岗位 -> 设置提醒 -> 投递岗位 -> 记录进度 -> 准备面试 -> Offer 管理
```

### 4.2 AI 闭环

```text
选择岗位 -> 简历匹配 -> 网申辅助 -> 模拟面试 -> 报告复盘 -> 针对性提升
```

### 4.3 运营闭环

```text
公众号/小红书获客 -> 小程序使用 -> 订阅提醒 -> 社群沉淀 -> 会员/服务转化
```

## 6. 开发阶段规划

### 6.1 第一阶段：留存与进度管理

目标：让用户每天有理由打开小程序，并能沉淀自己的求职数据。

功能范围：

- 求职进度管理。
- 岗位收藏与截止提醒。
- 校招/岗位截止提醒。
- 每日求职简报。
- 分享标题和分享图配置。

验收标准：

- 用户可以从职位详情加入求职进度。
- 用户可以在“我的”进入求职进度。
- 用户可以设置岗位截止时间和面试时间。
- 每日简报能汇总截止、面试、练习和建议。
- 分享配置统一管理，不破坏现有页面路径。

### 6.2 第二阶段：核心 AI 工具价值

目标：让用户围绕岗位完成简历优化、JD 匹配、网申材料和面试复盘。

功能范围：

- 简历中心能力增强。
- JD 匹配评分。
- 申请材料库。
- AI 网申助手。
- 面试错题本。
- AI 模拟面试报告体验优化。

验收标准：

- 用户能基于简历和岗位生成 JD 匹配结果。
- 用户能保存 Why Company、Why Role、Tell me about yourself 等材料。
- 用户能把面试薄弱题加入错题本。
- 所有 AI 生成内容展示 AI 免责声明。

### 6.3 第三阶段：内容资产沉淀

目标：让用户和运营共同沉淀面经、题库、STAR 模板和岗位内容资产。

功能范围：

- 面经投稿。
- 题库贡献。
- STAR 模板投稿。
- 内容审核后台。
- 热门内容榜单。
- 公司/岗位内容推荐。

验收标准：

- 投稿内容可进入后台审核。
- 审核通过后可展示到前端内容页。
- 支持敏感词、匿名、驳回原因和推荐位管理。

### 6.4 第四阶段：商业化与服务转化

目标：把 AI 能力、会员权益和求职服务形成可持续转化链路。

功能范围：

- 会员权益体系。
- 小程序虚拟支付。
- AI 次数包。
- 简历精修服务。
- 面试辅导服务。
- 求职机构线索转化。
- 邀请返利/分销。

验收标准：

- 支付环境为正式环境，Mock 支付关闭。
- 会员权益和功能权限一致。
- 订单、权益、到期时间和支付通知可追踪。
- 高风险支付链路单独测试与灰度。

## 11. 支付配置

当前生产环境使用小程序虚拟支付：

```text
PAYMENT_ENABLED=true
PAYMENT_PROVIDER=virtual
MEMBERSHIP_FEATURE_ENABLED=true
ENABLE_MOCK_PAYMENT=false
VIRTUAL_PAY_ENV=0
```

运行时已确认：

```text
provider=virtual
configured=true
mock=false
available=true
```

普通微信支付 JSAPI 相关变量当前未启用：

```text
WXPAY_MCH_ID
WXPAY_API_KEY
WXPAY_APP_ID
WXPAY_NOTIFY_URL
```

## 12. 数据与本地存储原则

2.0 中涉及的核心数据包括：

- 用户资料。
- 在线简历和 PDF 简历。
- 收藏岗位、公司、面经、校招。
- 求职进度记录。
- 截止提醒和面试提醒。
- JD 匹配报告。
- 申请材料。
- 面试错题。
- 会员订单和权益。

开发原则：

- 高价值数据优先同步到服务端。
- 弱联网场景可先写本地缓存，再做同步。
- AI 生成内容必须允许用户编辑、保存和删除。
- 简历、面试、语音、AI 对话等敏感数据需要清晰的删除入口和隐私说明。

## 17. 总结

职引 2.0 的关键不是增加更多入口，而是让用户形成稳定求职闭环：

```text
岗位闭环：找岗位 -> 收藏 -> 提醒 -> 投递 -> 进度追踪
AI 闭环：选岗位 -> 简历匹配 -> 网申辅助 -> 模拟面试 -> 报告复盘
运营闭环：内容获客 -> 小程序使用 -> 订阅提醒 -> 私域/会员转化
```

当这三个闭环跑起来，职引就能从“求职信息展示工具”升级为真正的“AI 求职管理助手”。
