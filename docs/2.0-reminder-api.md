# Job Reminder API

This is the implemented 2.0 reminder contract for saved jobs, job progress
deadlines, interviews, and later daily brief reminders.

## User APIs

- `GET /api/notify/reminders`
  - Auth: user token.
  - Query: `sourceType`, `reminderType`.
  - Returns current user's reminders, including disabled rows.

- `PUT /api/notify/reminders`
  - Auth: user token.
  - Upserts one reminder by `(userId, sourceType, targetId, reminderType)`.
  - Body:

```json
{
  "sourceType": "favorite_job",
  "targetId": "job_001",
  "reminderType": "deadline",
  "title": "Software Engineer",
  "company": "OpenAI",
  "jobTitle": "Software Engineer",
  "reminderDate": "2026-07-15",
  "reminderTime": "",
  "leadDays": [3, 1, 0],
  "enabled": true,
  "payload": {}
}
```

- `DELETE /api/notify/reminders/:sourceType/:targetId/:reminderType`
  - Auth: user token.
  - Disables the reminder instead of hard deleting it.

## Dispatch API

- `POST /api/notify/reminders/dispatch`
  - Auth: internal task auth with `X-Cron-Secret`.
  - Body/query: `date=YYYY-MM-DD` is optional; defaults to current
    Asia/Shanghai date.
  - Scans enabled reminders, sends station messages and subscription messages
    when the configured lead day matches.
  - Uses `sent_keys` to prevent duplicate sends for the same reminder/date/lead
    day.

Recommended cron: run once per day after 08:00 Asia/Shanghai.

Production helper:

```bash
cd /www/wwwroot/jobapp-server
npm run dispatch:reminders
```

Cron helper:

```cron
10 8 * * * /bin/bash /www/wwwroot/jobapp-server/scripts/run-reminder-dispatch.sh
```

## Mini Program Integration

- `miniprogram/utils/reminders.js` wraps reminder CRUD and subscription
  authorization.
- Favorites page writes `favorite_job` deadline reminders.
- Job progress page writes `job_progress` deadline and interview reminders.
