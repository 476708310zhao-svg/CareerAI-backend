# 求职线测试机器人使用文档

更新时间：2026-07-01

## 用途

求职线测试机器人用于一键验收小程序求职主链路和 2.0 求职线能力。它会自动启动本地后端、创建临时测试用户、执行接口和业务流程检查、生成 Markdown/JSON 报告，并在结束后清理测试数据。

## 常用命令

在后端项目目录执行：

```powershell
cd C:\Users\admin\Desktop\求职小程序\jobapp-server
```

基础验收：

```powershell
npm run bot:acceptance
```

求职线专项验收：

```powershell
npm run bot:career-line
```

3.0 开发者工具 UI 验收：

```powershell
npm run bot:e2e-3.0
```

3.0 UI 验收环境检查（不启动微信开发者工具）：

```powershell
npm run check:e2e-3.0
```

真实运行 3.0 UI 机器人前，需要先在微信开发者工具中开启 `设置 > 安全设置 > 服务端口`。如果该开关未开启，`npm run check:e2e-3.0` 会给出 warning，真实 E2E 会提前退出，避免长时间等待不可连接的自动化端口。

发布前完整验收：

```powershell
npm run check:acceptance
```

项目正式发布检查：

```powershell
npm run check:release
```

## 测试覆盖

### 基础链路

- 后端健康检查
- 功能开关
- 分享配置
- Banner
- 职位列表
- 职位详情
- 公司列表与公司详情
- 面经、校招、机构、资讯等内容入口
- 受保护接口匿名访问拦截
- 支付配置

### 用户链路

- 临时用户注册
- 登录
- 用户资料读取与更新
- 简历创建、详情读取、更新
- 收藏岗位新增、查询、取消
- 投递记录创建、追踪、列表读取、删除
- AI 参数校验，避免真实调用上游 AI
- Mock 支付下单、确认、核验

### 2.0 求职线专项

- 小程序求职线页面文件存在性检查
  - 求职进度
  - 每日求职简报
  - 申请材料库
  - 面试错题本
- 求职线工具模块存在性检查
  - `job-progress.js`
  - `reminders.js`
  - `application-materials.js`
  - `jd-match.js`
  - `interview-notebook.js`
- 申请材料库 CRUD
- JD 匹配报告创建、列表、详情
- 面试错题本新增、筛选、状态更新、删除
- 每日练习新增、列表、移除
- 求职提醒保存、列表、派发去重、关闭

### 3.0 开发者工具 UI 验收

新增 `scripts/e2e-3.0-bot.js`，通过微信开发者工具自动化运行 3.0 主链路，并生成 Markdown/JSON 报告和页面截图。

覆盖链路：

- 首页工作台打开
- 模拟登录态
- 附件简历上传
- AI 简历润色并采纳
- JD 匹配
- 保存投递到投递追踪
- 打开投递追踪详情
- 校验 JD 报告里的简历版本字段
- 面经题库 AI STAR 训练
- 训练稿保存到错题本
- AI 模拟面试生成报告
- 校招截止提醒订阅
- 会员额度页展示

注意：3.0 UI 机器人会 mock 微信授权、文件选择、订阅消息和后端请求，避免污染生产数据。它验证的是“微信开发者工具里的页面链路可走通”，不是物理手机真机自动点击。

## 外部环境验收

只读检查线上或预发环境：

```powershell
npm run bot:acceptance -- --base-url https://你的接口域名 --read-only
```

预发环境允许写测试数据时：

```powershell
npm run bot:acceptance -- --base-url https://预发接口域名 --write
```

如果要在外部环境测试提醒派发，需要额外提供内部任务密钥：

```powershell
$env:ACCEPTANCE_CRON_SECRET="你的 CRON_SECRET"
npm run bot:acceptance -- --base-url https://预发接口域名 --write
```

## 3.0 UI 自动化环境配置

首次运行前，需要确认微信开发者工具已安装，并允许命令行/自动化调用。Windows 常见 CLI 路径：

```text
C:\Program Files (x86)\Tencent\微信web开发者工具\cli.bat
```

如果自动识别失败，可以显式指定：

```powershell
npm run bot:e2e-3.0 -- --cli-path "C:\Program Files (x86)\Tencent\微信web开发者工具\cli.bat"
```

如果微信开发者工具已经开着，并且自动化端口冲突，可以先手动关闭开发者工具，或显式让机器人先退出旧实例再启动：

```powershell
npm run bot:e2e-3.0 -- --fresh-devtools
```

如果你已经通过微信开发者工具打开了自动化端点，也可以直接连接：

```powershell
npm run bot:e2e-3.0 -- --ws-endpoint ws://127.0.0.1:9420
```

也可以写入环境变量：

```powershell
$env:WECHAT_DEVTOOLS_CLI="C:\Program Files (x86)\Tencent\微信web开发者工具\cli.bat"
npm run bot:e2e-3.0
```

如果只想检查脚本和项目路径：

```powershell
npm run check:e2e-3.0
```

## 可选参数

```text
--base-url <url>       测试一个已经运行的环境
--write                对外部环境执行写入型验收
--read-only            只执行只读检查
--full-static          同时运行完整小程序发布静态检查
--skip-career-line     跳过 2.0 求职线专项写入检查
--skip-static          跳过轻量静态检查
--no-report            不生成报告
--report-dir <dir>     指定报告目录
--keep-data            保留机器人创建的测试数据
--timeout-ms <ms>      单个请求超时时间
--verbose              输出每一步详情
```

3.0 UI 机器人额外参数：

```text
--cli-path <path>       微信开发者工具 cli.bat 路径
--project-path <path>   小程序项目路径，默认 miniprogram/
--no-screenshot         不生成步骤截图
--keep-open             运行结束后不关闭微信开发者工具
--dry-run               只做环境检查，不启动开发者工具
--port <port>           指定微信开发者工具自动化端口
--fresh-devtools        启动前先退出已有微信开发者工具实例
--ws-endpoint <url>     连接已打开的自动化 WebSocket 端点
```

## 报告位置

默认生成在：

```text
C:\Users\admin\Desktop\求职小程序\jobapp-server\reports\acceptance\
```

每次会生成两个文件：

- `acceptance-时间戳.md`
- `acceptance-时间戳.json`

3.0 UI 验收报告默认生成在：

```text
C:\Users\admin\Desktop\求职小程序\jobapp-server\reports\e2e-3.0\
```

每次会生成：

- `e2e-3.0-时间戳.md`
- `e2e-3.0-时间戳.json`
- `screenshots-时间戳/` 步骤截图目录

看报告时重点关注：

- `failed` 是否为 `0`
- 是否有 `WARN`
- `career-line` 分组是否全部通过

## 判断标准

通过标准：

```text
Acceptance bot finished: xx passed, 0 failed
```

如果 `failed` 大于 0，需要先看失败行的分组：

- `static`：通常是小程序页面注册、文件缺失、发布配置问题
- `api`：通常是后端接口不可用或响应结构变更
- `security`：通常是鉴权边界异常
- `user-flow`：通常是登录、简历、收藏、投递、支付链路异常
- `career-line`：通常是 2.0 求职线材料库、JD 报告、错题本或提醒链路异常

## 注意事项

- 默认本地模式会自动清理测试用户和相关数据。
- 默认不会调用真实 AI 生成接口。
- 默认只走 Mock 支付，不会触发真实微信支付。
- 微信订阅消息授权弹窗属于小程序 UI 层，不在当前接口机器人覆盖范围内。
- 3.0 UI 机器人已覆盖微信开发者工具端到端页面链路，但真实手机上的微信登录授权、手机文件选择器、订阅消息授权弹窗和虚拟支付弹窗仍需人工真机确认。
