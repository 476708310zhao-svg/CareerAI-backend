# 职引 2.0 第二阶段接口契约草案

当前小程序端已用本地 service 层完成 P1 功能兜底，后续接后端时建议保持以下数据结构。

## 当前前端页面

- `package-ai/pages/application-materials/application-materials`：申请材料库，支持搜索、筛选、复制、编辑、删除。
- `package-ai/pages/interview-notebook/interview-notebook`：面试错题本，支持全部/每日练习/不会/已掌握筛选、再练一次、参考答案、移出复习清单。
- `package-ai/pages/ai-assistant/ai-assistant?action=application`：可直接打开网申助手抽屉生成草稿。

## 简历版本管理

现有前端已支持服务端多简历接口，未登录或接口不可用时继续使用 `onlineResume` 本地备份。

### 建议接口

- `GET /api/resumes`：获取当前用户简历版本列表。
- `GET /api/resumes/:id`：获取单个简历版本详情。
- `POST /api/resumes`：创建简历版本。
- `PUT /api/resumes/:id`：更新简历版本名称和内容。
- `DELETE /api/resumes/:id`：删除简历版本，至少保留 1 份。

## 后端接口前缀

第二阶段资产接口已落地在：

```text
/api/career-assets
```

旧草案中的独立路径可视为产品命名，实际实现统一走该前缀。

## JD 匹配报告

当前前端本地存储键：`jdMatchReports`。

### 数据字段

```json
{
  "id": "match_001",
  "jobId": "job_001",
  "jobTitle": "Software Engineer Intern",
  "company": "Google",
  "resumeName": "当前在线简历",
  "score": 82,
  "matchedKeywords": ["Python", "SQL"],
  "missingKeywords": ["AWS", "Docker"],
  "projectSuggestion": "推荐强化的项目经历",
  "atsRisk": "中",
  "suggestions": ["补充关键词", "量化项目结果"],
  "createdAt": "2026-07-01T10:00:00.000Z"
}
```

### 已实现接口

- `POST /api/career-assets/jd-match-reports`：保存匹配报告。
- `GET /api/career-assets/jd-match-reports`：获取历史报告，支持 `jobId`、`resumeName`。
- `GET /api/career-assets/jd-match-reports/:id`：获取报告详情，支持服务端 id 或本地 client id。

## AI 网申材料库

当前前端本地存储键：`applicationMaterials`。

### 数据字段

```json
{
  "id": "material_001",
  "questionType": "why_company",
  "questionLabel": "Why this company?",
  "jobId": "progress_001",
  "company": "Google",
  "jobTitle": "Software Engineer Intern",
  "content": "回答草稿正文",
  "createdAt": "2026-07-01T10:00:00.000Z"
}
```

### 问题类型

`why_company`、`why_role`、`tell_me`、`project`、`leadership`、`conflict`、`strength`、`career_goal`、`work_auth`、`relocation`。

### 已实现接口

- `GET /api/career-assets/application-materials`：获取材料库列表，支持 `jobId`、`questionType`。
- `POST /api/career-assets/application-materials`：保存草稿，支持本地 client id 幂等同步。
- `PUT /api/career-assets/application-materials/:id`：编辑草稿，支持服务端 id 或本地 client id。
- `DELETE /api/career-assets/application-materials/:id`：删除草稿。
- 草稿生成仍在小程序本地 service/AI 助手完成，后续可再补 `generate` 服务端接口。

## 面试错题本

当前前端本地存储键：`interviewMistakeNotebook`、`dailyPracticeQuestions`。

### 数据字段

```json
{
  "id": "q_001",
  "title": "请介绍一次你处理冲突的经历",
  "answer": "参考答案",
  "category": "behavior",
  "difficulty": "中等",
  "status": "unknown",
  "createdAt": "2026-07-01T10:00:00.000Z",
  "updatedAt": "2026-07-01T10:00:00.000Z"
}
```

### 状态枚举

`unknown` 不会，`mastered` 已掌握，`saved` 已保存。

### 已实现接口

- `GET /api/career-assets/interview-notebook`：获取错题本列表，支持 `status`、`category`。
- `POST /api/career-assets/interview-notebook`：新增或更新错题。
- `PUT /api/career-assets/interview-notebook/:id/status`：更新掌握状态。
- `DELETE /api/career-assets/interview-notebook/:id`：移除错题。
- `GET /api/career-assets/interview-daily-practice`：获取每日练习题。
- `POST /api/career-assets/interview-daily-practice`：加入每日练习。
- `DELETE /api/career-assets/interview-daily-practice/:id`：移出每日练习。
