# 页面级 UI/UX 验收清单

> 适用范围：微信小程序所有页面的设计、交互、性能和代码维护验收。
> 使用方式：每次优化页面前先对照检查，优化后逐项打勾；未通过项进入下一轮优化任务。

## 一、全局视觉验收

- [ ] 页面背景统一为白色或极浅灰 AI SaaS 风格，不出现廉价渐变、杂色底或突兀暗色块。
- [ ] 字号层级清晰：页面标题、模块标题、正文、辅助信息、标签文字有明显层级。
- [ ] 主色使用 `--color-brand` / `--brand`，不新增随意蓝紫色值。
- [ ] 卡片使用统一圆角、轻边框、克制阴影，不出现厚重阴影或多套卡片风格。
- [ ] 输入框、按钮、标签、底部操作栏优先复用公共组件或 `ux-*` 工具类。
- [ ] 图标风格统一为线性、简洁、高级，不混用卡通、emoji、低清位图。
- [ ] 页面底部安全区正常，不出现过大空白、被 tabbar 遮挡或横向滚动条。

## 二、移动端布局验收

- [ ] iPhone 小屏宽度下文字不挤压、不换成异常两行、不横向溢出。
- [ ] 顶部导航、搜索栏、tabs、筛选 chips 在真机/开发者工具中对齐稳定。
- [ ] 列表卡片左右边距统一，卡片之间留白节奏一致。
- [ ] 固定底部操作栏使用 `ux-bottom-bar` 或公共 action bar，并预留安全区。
- [ ] 可点击区域高度不低于 64rpx，主按钮建议 76rpx-88rpx。
- [ ] 图片和 logo 使用 `c-company-logo` 或统一图片规范，加载失败有兜底。

## 三、UX 路径验收

- [ ] 首屏能在 3 秒内让用户知道“这个页面能做什么”。
- [ ] 每个核心页面只保留一个最强主 CTA，其它操作降级为次按钮、图标或入口卡。
- [ ] 空状态必须给下一步行动，不只展示“暂无数据”。
- [ ] Loading 状态有明确反馈，AI 场景要说明正在理解、匹配或生成。
- [ ] 筛选、tab、列表刷新、分页加载都有清晰反馈，不让用户误以为卡死。
- [ ] 页面跳转路径可回退，用户不会迷失在多层流程中。
- [ ] 关键商业转化入口不打扰，但在强意图场景清晰出现。

## 四、AI 产品体验验收

- [ ] AI 页面表达专业、克制、可信，不使用过度花哨动画。
- [ ] AI 回复结果需要有可行动入口：复制、保存、继续追问、拆成清单或进入相关功能。
- [ ] AI 等待状态使用轻量阶段反馈，避免只有 spinner。
- [ ] AI 结果卡优先复用 `ux-ai-result-card` / `ux-result-action`。
- [ ] AI 推荐卡片要说明推荐理由或下一步，不只给标题。
- [ ] AI 错误状态要能重试，并保留用户输入。

## 五、性能验收

- [ ] 非首屏图片使用懒加载，首屏关键 banner/logo 保持即时加载。
- [ ] 长列表有分页、加载更多、无更多状态，不一次性渲染过多数据。
- [ ] 避免重复请求和过期响应覆盖当前页面状态。
- [ ] 高频交互避免连续 `setData` 大对象。
- [ ] 页面动效只使用 opacity/transform 等 GPU 友好属性。
- [ ] 图片资源走 HTTPS，后台上传图片应压缩后再展示。

## 六、代码维护验收

- [ ] 新样式优先使用 `app.wxss` token 或已有 `ux-*` 工具类。
- [ ] 页面内不新增重复按钮、卡片、标签、logo 样式，优先复用公共组件。
- [ ] 新组件命名遵循 `c-*`，工具类命名遵循 `ux-*`。
- [ ] 页面 WXSS 不保留阶段性注释、乱码注释、废弃覆盖。
- [ ] 不引入与当前任务无关的文件改动。
- [ ] 每轮优化必须更新路线图并提交 GitHub。

## 七、重点页面专项验收

### 首页

- [ ] 首屏清楚呈现产品价值、档案完善入口和 AI 助手入口。
- [ ] 金刚区图标统一，高频功能不超过 8 个。
- [ ] 热门企业、热门岗位、求职快讯层级清晰，卡片高度不过长。
- [ ] 岗位为空时引导完善档案或搜索岗位。

### 岗位推荐页

- [ ] 标题使用“岗位推荐”，搜索、筛选、热门 chips 不拥挤。
- [ ] 岗位 logo 与公司详情/职位详情保持一致。
- [ ] 列表模式和地图模式切换反馈明确。
- [ ] 收藏、投递、AI 面试等操作层级清晰。

### 职位详情页

- [ ] 底部主 CTA 以“一键投递”为主，AI 面试和分享为次级操作。
- [ ] 公司 logo 使用 `c-company-logo`。
- [ ] 标签使用 `c-tag`，地点、类型、发布时间、签证支持视觉统一。
- [ ] 求职规划入口不抢主 CTA，但能形成闭环。

### 公司详情页

- [ ] 公司 logo 使用 `c-company-logo`，失败兜底稳定。
- [ ] 职位、面经、薪资、AI 洞察卡片使用 `ux-list-card`。
- [ ] 底部主 CTA 指向“查看在招岗位”，收藏/分享降为图标。
- [ ] AI 洞察内容可读，有明确刷新或生成反馈。

### AI 助手页

- [ ] 页面整体保持白色 ChatGPT 风格，不使用深色大面积背景。
- [ ] 历史侧栏、常用功能、最近对话层级清晰。
- [ ] AI 回复后的结果卡使用 `ux-ai-result-card`。
- [ ] 输入栏固定稳定，键盘弹起不遮挡消息。

### 我的页

- [ ] 用户信息、会员入口、求职工具、设置入口层级清楚。
- [ ] 底部留白合理，不出现横向滚动条。
- [ ] 会员 CTA 不喧宾夺主，但在用户有求职意图时清晰可见。
