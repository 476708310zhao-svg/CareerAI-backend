# 职引小程序 3.0 优化开发路线图

## 1. 产品目标

将小程序从“求职工具合集”升级为高频打开的 **AI 求职助手 + 校招日历 + 投递管理工具**。

官网负责完整链路：上传简历、生成报告、导出文档、复杂编辑。小程序负责高频轻动作：看校招、快速分析 JD、收藏岗位、查投递进度、看提醒、做轻量 AI 优化。

核心目标：

1. 用户每天有理由打开。
2. 用户可以快速完成一个求职动作。
3. 复杂 AI 简历优化被拆成小程序可操作的轻流程。
4. 强化校招日历、岗位提醒、面经题库等高频场景。
5. 与官网账号、简历、投递记录、报告数据结构打通。

## 2. P0 开发范围

P0 先完成“每日工作台 + JD 匹配 + 投递追踪 + 校招筛选”的最小闭环。

| 模块 | 目标 | 关键交付 |
| --- | --- | --- |
| 首页工作台 | 替代功能堆叠，给出今日建议 | 最近 ATS 分数、今日校招数、待处理投递、AI 下一步 |
| JD 快速匹配 | 1 分钟判断岗位是否值得投 | JD/简历输入、匹配分、关键词、优化建议、保存投递 |
| 投递追踪基础版 | 让用户每天查看求职进度 | 添加记录、修改状态、下一步事项、关联 JD |
| 校招日历筛选 | 放大现有校招优势 | 届别、类型、地区、行业、学历、留学生友好筛选 |
| 我的简历/报告 | 统一后续数据入口 | 历史简历、ATS 报告、跳转官网完整优化 |

## 4. 数据模型草案

### JD 匹配记录

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string/number | 匹配记录 ID |
| `user_id` | string/number | 用户 ID |

| `resume_text` | text | 简历文本或快照 |

| `created_at` | datetime | 创建时间 |

### 简历评分记录

评分维度：ATS 格式、关键词覆盖、经历表达、项目量化、动词使用、技能结构、教育背景、可读性。

### 简历版本

| 字段 | 类型 | 说明 |

| `resume_version_id` | string/number | 版本 ID |

| `target_role` | string | 目标岗位 |
| `target_company` | string | 目标公司 |
| `ats_score` | number | ATS 分数 |
| `last_used_at` | datetime | 最近使用时间 |
| `created_at` | datetime | 创建时间 |
| `source_jd_id` | string/number | 来源 JD 分析 |

### 投递记录

状态流转：已收藏 -> 已投递 -> OA -> 一面 -> 二面 -> 终面 -> Offer / Rejected。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `company_name` | string | 公司名称 |
| `job_title` | string | 岗位名称 |
| `job_url` | string | 岗位链接 |
| `applied_at` | datetime | 投递时间 |
| `status` | enum | 当前状态 |
| `resume_version_id` | string/number | 使用的简历版本 |
| `next_action` | string | 下一步事项 |
| `interview_time` | datetime | 面试时间 |
| `note` | text | 备注 |

## 5. 首页工作台设计

首页首屏目标是直接回答“我今天该做什么”。

结构：

```text
顶部：
今日求职进度 / AI 一句建议

核心卡片：
1. 上传 JD，测匹配度
2. 今日校招日历
3. 我的投递进度
4. AI 模拟面试

快捷入口：
AI 简历 / 校招日历 / 面经题库 / 求职助手 / 我的
```

首屏文案方向：

```text
今天适合投递 3 个岗位
你的简历平均匹配分：76
建议优先优化 Data Analyst 方向简历
```

未登录用户展示默认引导卡片；已登录用户展示最近 ATS 分、今日新增校招数、待处理投递事项和 AI 推荐动作。

## 6. JD 快速匹配流程

```text
粘贴 JD
↓
选择已有简历 / 粘贴简历
↓
生成匹配分
↓
查看缺失关键词
↓
生成优化建议
↓
保存为投递记录
```

页面展示：

1. 匹配分。
2. 岗位关键词。
3. 缺失关键词。
4. 是否建议投递。
5. 简历优化建议。
6. 面试准备建议。

CTA：

1. 保存岗位。
2. 生成优化简历。
3. 加入投递追踪。
4. 去官网导出完整版简历。

## 7. 校招日历优化

筛选项：

| 维度 | 选项 |
| --- | --- |
| 届别 | 2026 / 2027 / 2028 |
| 类型 | 校招 / 暑期实习 / 提前批 / 日常实习 |
| 地区 | 中国 / 北美 / 英国 / 新加坡 |
| 行业 | 互联网 / 金融 / 咨询 / 快消 / AI / 硬件 |

| 留学生 | 是否支持留学生 |

推荐逻辑：专业、目标地区、求职方向、简历关键词、历史收藏岗位。

## 8. 面经题库优化

目标：从内容列表升级为面试准备工具。

功能：

1. 公司、岗位、难度筛选。
2. AI 生成回答框架。
3. STAR 回答生成。
4. 模拟追问。
5. 收藏到面试清单。
6. 练习记录和面试复盘。

题目详情页 CTA：

```text
AI 帮我回答
生成 STAR 案例
模拟面试追问
收藏到我的面试清单
```

## 9. AI 求职助手工作流化

入口问题模板：

```text
我想优化简历
我想分析一个岗位
我想准备面试
我想比较 Offer
我想规划求职方向
我想找适合我的校招
```

每个入口进入对应工作流，而不是直接闲聊。例如“我想分析一个岗位”进入 JD 粘贴、选择简历、生成岗位分析、保存到投递记录。

## 10. 会员体系

免费用户：

1. 每日 1 次 JD 快速匹配。
2. 每月 1 次简历评分。
3. 可收藏 10 个岗位。
4. 可查看基础校招信息。

会员用户：

1. 无限 JD 匹配。
2. 无限简历评分。
3. 无限投递追踪。
4. AI 简历优化。
5. AI 面试复盘。
6. 校招截止提醒。
7. 高级筛选。
8. 留学生友好岗位标签。
9. 优先查看精选岗位。

付费触发点：完整匹配报告、全部缺失关键词、优化版简历、面试复盘导出、多个投递提醒。

## 12. 验收标准

1. 用户进入首页后，能看到明确的今日求职建议。
2. 用户可以在 1 分钟内完成一次 JD 快速匹配。
3. 匹配结果必须有分数、关键词、缺失项、下一步建议。
4. 用户可以把 JD 匹配结果保存为投递记录。
5. 校招日历必须支持筛选、收藏和提醒。
6. 面经题库不能只是内容列表，需要接入 AI 回答、STAR 生成和模拟追问。
7. 小程序和官网的数据结构统一，后续能打通账号、简历、报告和投递记录。
