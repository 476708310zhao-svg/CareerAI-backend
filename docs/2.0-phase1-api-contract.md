# 职引 2.0 第一阶段接口契约草案

当前小程序端已先用本地存储封装 service 层，后续接后端时建议保持以下数据结构。

## 求职进度

### 数据字段

```json
{
  "id": "progress_001",
  "sourceJobId": "job_001",
  "company": "Google",
  "jobTitle": "Software Engineer Intern",
  "city": "Mountain View",
  "salary": "$8k/mo",
  "jobLink": "https://...",
  "appliedAt": "2026-07-01",
  "deadline": "2026-07-15",
  "interviewTime": "2026-07-20 10:00",
  "status": "online_apply",
  "notes": "HR 邮件已回复",
  "resumeVersionId": "resume_data_v3",
  "interviewQuestionIds": ["q_001"],
  "aiReportId": "report_001",
  "reminderEnabled": true,
  "reminderLeadDays": [3, 1]
}
```

### 状态枚举

`collected` 已收藏，`applied` 已投递，`online_apply` 网申中，`oa` OA 阶段，`first_interview` 一面，`second_interview` 二面，`hr_interview` HR 面，`offer` Offer，`rejected` 拒信，`closed` 已结束。

### 建议接口

- `GET /api/job-progress`：获取当前用户求职进度列表，支持 `status`、`keyword`。
- `POST /api/job-progress`：新增求职进度。
- `PUT /api/job-progress/:id`：更新状态、备注、时间、关联简历等字段。
- `DELETE /api/job-progress/:id`：删除记录。

## 收藏岗位提醒

- `PUT /api/favorites/:type/:targetId/reminder`：设置收藏对象提醒。
- 请求体：`{ "deadline": "2026-07-15", "reminderEnabled": true, "reminderLeadDays": [3, 1] }`
- 当前前端已在 `favorites.js` 本地记录 `deadline`、`reminderEnabled`、`reminderLeadDays`。

## 每日求职简报

- `GET /api/daily-brief`：返回今日推荐、即将截止、进度汇总、面试任务、每日题目、AI 建议、热门快讯。
- 当前前端可从本地进度、收藏、缓存推荐岗位和快讯生成兜底简报。
